# QueteAPI 开发设计

这是一款Web API应用，名字叫做Quate API，用于为网站、页面提供几句话文本的语句，比如一些诗句、句子等等，每次API调用随机返回一条。要做成多用户、轻量化、前端设计现代化且简约的应用，最好使用Docker可以一键部署。

## 主要功能

1. 多用户可以进行注册登录（默认用户为管理员，账号admin，密码admin），管理员支持简单的用户管理。

2. 每个用户可以创建自己的语句仓库，并且支持往里面便捷添加想要加入的语句（支持换行）。仓库只能由创建者添加、修改、删除语句。所有的仓库页面默认开放访问，并提供API，每一个仓库以及每一个语句都有一个简约页面。

3. API调用时需要携带仓库名，用于请求对应的语句仓库，然后API随机返回仓库内的一条文本，以及这条语句对应的链接（JSON格式）。

4. 仓库界面可以显示每个语句的使用次数、对应页面访问次数、整个仓库调用次数，创建者还在这个界面可以查看到请求来源的网站、请求日志等详细信息。

5. 添加一些其他的必要功能。

## 要求

1. 前端界面以白色为底，设计简约、高端，任何地方都不要使用紫色、深蓝色等颜色，不要花哨，但是设计要高端、现代（模仿苹果网站等设计），语言都使用中文，禁止往前端添加任何表情符号！可以使用一些前端库，高级一些。

2. 整个项目尽可能简约、便于部署，部署最好用Docker/Docker Compose一键部署

3. 代码需要做好分层设计，每个文件不能超过100行，使用面向对象等设计，提升扩展性，高内聚低耦合。

4. 数据库要使用最方便、性能最高、部署最简单、占用最少的方案

5. 完成之后在根目录完成使用与部署文档
